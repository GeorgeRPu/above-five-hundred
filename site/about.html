How these forecasts work

Every number on this site is produced by an open statistical model, published as plain JSON, and rendered in the browser. No black boxes.

The pipeline

Each model is a program (most are Python) that ingests game results, computes ratings and simulates the remainder of the season, then writes its output to site/data/<model-slug>/latest.json. The site is fully static: pages fetch that JSON and draw the tables, win-probability bars and sparklines you see. Re-running a model and pushing the new JSON is all it takes to update a forecast.

Reading the numbers

Ratings are Elo-style power scores: the gap between two teams' ratings maps to a win probability for a game between them, with an adjustment for home advantage. Playoff and title odds come from Monte Carlo simulation — the model plays out the rest of the season thousands of times and counts how often each outcome occurs. Probabilities are estimates, not promises: a 75% favorite should lose one time in four.

Publishing your own model

Add an entry to site/data/models.json and write a latest.json matching the schema documented in the repository README. The sample generator in scripts/generate_sample_data.py is a working end-to-end example.
